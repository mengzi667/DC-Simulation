# 仿真机制

1.FG outboud:

两个过程：一个是处理货物，这之后是货物出库

处理货物有数量上限：根据pallet/FTE * FTE计算，opening hour影响FTE
处理货物的时间通过 总pallet数量/工作效率计算

出库占用一个timeslot，也要占用FTE

数据格式：
order id | 生成 time | 送走 time | pallets | region | 
对于region是G2的：送走 time 50%是当天，50%的次日
对于region是ROW的：送走 time全都是次日
G2和ROW是8:2

2.R&P outbound:不要用九月之后的数据

两个过程：一个是处理货物，这之后是货物出库
数据格式：
order id | 生成 time | pallets |

处理货物有数量上限：根据pallet/FTE * FTE计算，opening hour影响FTE
处理货物的时间通过 总pallet数量/工作效率计算

出库占用一个timeslot，也要占用FTE

3.R&P和FG inbound:
两个过程：一个是货物入库，这之后是处理货物
进来，24小时内处理货物完成

处理货物有数量上限：根据pallet/FTE * FTE计算，opening hour影响FTE
处理货物的时间通过 总pallet数量/工作效率计算

数据格式：
order id | 生成 time | pallets |

入库占用一个timeslot，也要占用FTE，对于有送走时间要求的订单，尽量出库的timeslot在第二天的上午（这是一个直觉上的建议）

超时就延期并记录
